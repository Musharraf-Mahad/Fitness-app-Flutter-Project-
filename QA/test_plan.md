# FitTrack – Test Plan

## 1. Introduction

This document describes the testing strategy for the **FitTrack Mobile Application**. The purpose of this test plan is to ensure that the application functions correctly, meets user requirements, and provides a reliable user experience.

The testing process verifies the functionality, usability, and performance of the application before final delivery.

---

## 2. Testing Objectives

The objectives of testing are:

* Verify that all features work as expected
* Identify bugs or unexpected behaviors
* Ensure the user interface works correctly
* Validate navigation between screens
* Confirm that user data is handled correctly

---

## 3. Scope of Testing

### Features to be Tested

The following components of the FitTrack application will be tested:

1. **User Authentication**

   * User registration
   * User login
   * Input validation

2. **Dashboard**

   * Display of daily steps
   * Calories burned
   * Distance tracking
   * Navigation to other sections

3. **Step Tracker**

   * Display of step progress
   * Goal tracking
   * Activity visualization

4. **Workout Screen**

   * Start workout session
   * Stop workout session
   * Display workout duration

5. **Goal Management**

   * Set daily step goal
   * Display goal progress

6. **Profile Screen**

   * View user profile
   * Edit profile information
   * Logout functionality

---

## 4. Types of Testing

### Functional Testing

Ensures each feature works according to the requirements.

### User Interface Testing

Verifies that the UI elements appear correctly on different screen sizes and devices.

### Navigation Testing

Checks that all navigation buttons and links lead to the correct screens.

### Performance Testing

Ensures that the application loads and responds efficiently.

### Usability Testing

Evaluates how easy it is for users to interact with the application.

## 7. Risks and Assumptions

### Risks

* Step tracking may depend on device sensors
* UI behavior may vary across devices

### Assumptions

* The application runs on Android devices
* Users have a stable device environment


## 8. Conclusion

The testing process ensures that the FitTrack application performs reliably and meets the expected functionality. Proper testing helps identify and resolve issues before final deployment.
